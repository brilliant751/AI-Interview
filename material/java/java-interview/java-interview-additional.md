## 第1 题：JVM 内存结构与对象生命周。
### 题干

请说JVM 运行时内存结构，并结合一个对象从创建到回收的全过程解释关键机制。
### 类别

技术
### 解析

可按“线程私有（程序计数器、虚拟机栈、本地方法栈 线程共享（堆、方法区/元空间）”来回答。对象通常Eden 分配，经Minor GC 后可能进Survivor，再晋升老年代。判断回收可达性主要依赖可达性分析而非引用计数，常见回收器会结合分代和并发标记来平衡吞吐与停顿。

## 第2 题：G1 CMS 的区别和选型

### 题干

G1 CMS 的核心差异是什么？在什么场景下你会优先选择 G1。
### 类别

技术
### 解析

CMS 以最短停顿为目标，老年代采用并发标记清理，但会产生碎片且已被移除。G1 Region 管理堆，支持可预测停顿目标，能做整堆回收并减少碎片。JDK 11+ 的生产环境通常优先 G1，除非有明确压测数据证明其他收集器更优。

## 第3 题：类加载机制与双亲委派

### 题干

解释 Java 类加载过程和双亲委派模型，为什么它能提升安全性？

### 类别

技术
### 解析

类加载包括加载、验证、准备、解析、初始化。双亲委派保证核心类优先由引导类加载器加载，避免业务代码伪java.* 类造成安全问题。若要打破委派（如 SPI、热部署），需要明确边界并控制类冲突。

## 第4 题：volatile 的可见性与有序。
### 题干

volatile 能解决哪些并发问题，不能解决哪些问题。
### 类别

技术
### 解析

volatile 保证写后读可见性，并通过内存屏障限制特定重排序。它不保证复合操作原子性，因此 count++ 仍可能并发出错。适合状态标记、配置刷新，不适合需要互斥更新的场景。
---

## 第5 题：AQS 原理与常见实。
### 题干

AQS 是如何支ReentrantLock、CountDownLatch、Semaphore 的？

### 类别

技术
### 解析

AQS 维护一state CLH 队列，失败线程入队并阻塞，释放时按规则唤醒。不同同步器本质state 语义不同：锁是重入计数，闭锁是倒计时，信号量是许可数。面试回答要突出“模板方+ 队列 + CAS”。
---

## 第6 题：ThreadLocal 内存泄漏问题

### 题干

ThreadLocal 为什么可能导致内存泄漏？如何规避。
### 类别

技术
### 解析

ThreadLocalMap key 是弱引用，value 是强引用；key 被回收后若线程长期存活，value 可能滞留。在线程池场景尤其明显。规避方式是 finally remove，避免在可复用线程里长期持有大对象。
---

## 第7 题：HashMap 1.7 1.8 的关键差。
### 题干

HashMap JDK 1.8 相比 1.7 做了哪些关键改进。
### 类别

技术
### 解析

1.8 引入链表转红黑树（满足阈值）降低极端冲突查询复杂度，并优化扩容迁移逻辑.7 头插法在并发误用下可能形成环链，1.8 改为尾插后该风险降低。核心结论是：HashMap 仍非线程安全，多线程应选并发容器。
---

## 第8 题：ConcurrentHashMap 的并发控。
### 题干

ConcurrentHashMap JDK 1.8 是如何实现高并发读写的？

### 类别

技术
### 解析

1.8 取消 Segment，采用数+ 链表/红黑树结构，基于 CAS synchronized 细粒度控制桶级并发。读操作尽量无锁，写操作只锁定冲突桶。扩容时通过多线程协助迁移提升吞吐。

## 第9 题：TCP 三次握手与四次挥。
### 题干

为什么建立连接是三次握手，断开连接通常是四次挥手？

### 类别

技术
### 解析

三次握手用于双方确认收发能力并同步序列号，避免历史连接请求污染。挥手阶TCP 全双工，双方关闭发送方向通常分两步，因此常见四次。TIME_WAIT 作用是确保对端收到最ACK 并清理网络残包。

## 第10 题：HTTP/1.1、HTTP/2、HTTP/3 对比

### 题干

从性能和传输层角度，对HTTP/1.1、HTTP/2、HTTP/3。
### 类别

技术
### 解析

HTTP/1.1 有队头阻塞与连接复用不足问题；HTTP/2 引入二进制分帧、多路复用、头部压缩，但仍基于 TCP，丢包会影响同连接流。HTTP/3 基于 QUIC（UDP）在传输层降低队头阻塞并改进弱网体验。生产选型需结合网关和客户端支持情况。

## 第11 题：MySQL 事务隔离MVCC

### 题干

InnoDB 是如何用 MVCC 支撑可重复读的？

### 类别

技术
### 解析

MVCC 通过隐藏版本列和 undo log 构建一致性视图，读已提交与可重复读的区别在于快照生成时机。可重复读可避免不可重复读，并在 InnoDB 中通过 next-key lock 抑制大多数幻读。回答时建议区分快照读与当前读。
---

## 第12 题：MySQL 索引失效常见原因

### 题干

哪些写法会导致索引失效？如何SQL 改写。
### 类别

技术
### 解析

## 第13 题：为什MySQL 常用 B+Tree

### 题干

数据库索引为什么常B+Tree，而不是红黑树或哈希？

### 类别

技术
### 解析

B+Tree 高扇出、树高低，磁I/O 次数少，且天然支持范围查询和排序。红黑树层高更高不适合磁盘页访问，哈希不支持有序范围扫描。数据库场景的核心约束是磁盘访问代价而非单次比较。
---

## 第14 题：Redis RDB AOF 取舍

### 题干

RDB AOF 的优缺点分别是什么，生产如何配置。
### 类别

技术
### 解析

RDB 恢复快、文件紧凑但可能丢失最近一次快照后的数据；AOF 持久性更好但文件更大、恢复相对慢。常见做法是两者结合：主用 AOF（everysec），同时保留 RDB 作为全量备份与快速恢复手段。
---
