# AI模拟面试

## 主线
- 用户上传简历
- 用户开始面试流程
- 面试环节
- 面试结束，生成面试报告
- 用户的下一次面试将参考之前的面试记录和报告进行

## 面试单位循环：
- 输入：语音转文字（FunASR）
- 回答：LLM（deepseek-r1:8b）
- 输出：文字转语音（PaddleSpeech）

## 基建

### 题库

- 使用SQLite
- >=150 x 2 道题

### 知识库

#### Chroma
- >=200 x 2 条知识记录
- LangChain提供支持
- 向量数据库

1、索引
- 文本分割
    - 使用qwen3.5-2b（ollama）
- 文本嵌入
    - 使用nomic-embed-text（ollama）
- 存储
    - Chroma数据库

2、检索
- 向量检索（Chroma）
- 相关性排序（LLM）
- TopK，返回前K条结果

3、生成
#### 本地缓存
- 用户简历 + 自我介绍 + 用户所有回答
#### 开始阶段
- 等待用户自我介绍完成

#### 项目提问
- 根据用户的简历和自我介绍，提问项目经历
- 根据用户的回答，提问技术细节

#### 技术基础提问
- 根据设定的难度系数，在知识库中检索相关知识点，进行提问
- 根据用户的回答，正确则根据面试规则决定是否深入提问，错误则当场指出错误并进行讲解，并不再追问
- 提问3-5题即可

#### 行为面试提问
- 根据设定的难度系数，在题库中的各类型题抽取一题，进行提问
- 根据用户回答和题目类型决定是否追问
- 场景题等开放题则根据用户的回答进行追问，直到用户无法继续回答或逻辑链路完整或超过3次追问为止

#### 结束阶段
- 先结束面试进程
- 异步生成面试报告，包含用户的回答、面试官的评价和建议等内容
- 面试的评估与报告生成由另一个agent负责，输入为面试中的所有记录和相关信息
