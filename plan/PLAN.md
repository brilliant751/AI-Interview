# AI模拟面试与能力提升软件-项目计划书

## 1. 项目目标与范围
- 项目名称：面向计算机相关专业学生的 AI 模拟面试与能力提升平台
- 目标用户：高校计算机相关专业学生（重点覆盖毕业求职阶段）
- 建设目标：
  - 提供岗位化、多轮、多模态模拟面试能力
  - 提供可量化、可追踪、可解释的能力评估报告
  - 形成“练习-评估-提升-复盘”的闭环学习路径
- 首期岗位范围（至少 2 个，满足赛题要求）：
  - Java 后端开发工程师
  - Web 前端开发工程师

## 2. 具体技术栈选型

### 2.0 运行环境版本基线（可执行矩阵）
- Node.js：>= `20.11`（建议锁定 `20.19.x LTS`）
- pnpm：>= `10.0.0`（建议锁定 `10.6.x`）
- npm（仅在不使用 pnpm 时）：>= `10.5.0`
- Python：>= `3.10`（建议锁定 `3.11.x`）
- PostgreSQL：>= `15`（建议锁定 `16.x`）
- 说明：
  - 前端与工具链统一使用 Node 20 LTS，避免本地与 CI 环境差异。
  - 包管理器默认采用 pnpm；npm 仅作为降级选项保留最低版本要求。
  - 数据库基线设为 PostgreSQL 15+，兼顾 pgvector 能力与生产可用性。

### 2.1 前端
- React 18
- TypeScript 5.x（锁定到 `5.9.x` minor，避免跨 minor 行为差异）
- Vite
- React Router
- Zustand
- Axios
- Monaco Editor（在线代码编辑器）
- 关键前端能力：
  - 面试对话页（文本/语音输入）
  - 在线编程面板（题目描述、语言选择、代码编辑、运行与结果回显）
  - 实时状态管理（会话状态、计时、问题轮次、得分草稿）
  - 评估报告可视化（雷达图、趋势图、题型分布）
  - 历史记录与岗位切换
  - 登录态管理（鉴权、会话续期、权限路由）

### 2.2 后端
- Python 3.10
- FastAPI
- 模块划分：
  - `interview-service`：会话编排、追问策略、评分汇总、计时策略
  - `rag-service`：题库检索、知识库检索、上下文构建
  - `analysis-service`：内容分析、表达分析、报告生成、成长预测
  - `auth-service`：用户注册登录、令牌签发、权限校验
  - `code-runner-service`：在线代码提交、运行状态轮询、结果标准化
  - `evaluation-orchestrator`：多 Agent 评估编排、证据链聚合、综合评分计算
  - `model-gateway`：统一封装 LLM/ASR/TTS 调用与降级

### 2.3 数据层
- PostgreSQL：业务主库（用户、岗位、题库、会话、评估报告、代码提交、评估证据链、预测结果）
- pgvector：向量检索（知识片段、题目语义索引、优秀回答示例）
- Redis：会话缓存、限流、短期上下文、异步任务状态、计时告警状态

### 2.4 部署
- Docker + Docker Compose
- 容器建议：
  - `frontend`、`backend-api`、`worker`
  - `postgres`（启用 pgvector）
  - `redis`
  - `asr-local`（本地语音识别服务）
  - `tts-local`（本地语音合成服务）

## 3. 系统总体架构（简版）
- 前端负责交互与可视化，后端统一编排对话与评估流程
- RAG 从“岗位题库 + 本地知识库”召回上下文，提供给 LLM 进行追问和打分解释
- ASR 将语音转文本；TTS 负责 AI 面试官语音播报
- 在线编程链路由 `code-runner-service` 统一调用代码执行引擎，返回编译/运行结果、资源消耗、用时数据
- 面试完成后由 `evaluation-orchestrator` 调用多个评估 Agent，输出分维度评分、证据链与综合结论
- 分析模块基于历史序列生成结构化报告、通过概率区间与能力差距预警清单，并写入数据库

## 4. 需要完成的功能点

### 4.1 岗位化题库与知识库
- 岗位配置管理：
  - 支持岗位管理CRUD，上线/下线
  - 每岗位独立评分维度权重（技术深度/表达逻辑/岗位匹配等）
- 题库建设（每岗位不少于 150 题，覆盖四类）：
  - 技术知识题
  - 项目经历深挖题
  - 场景设计题
  - 行为与沟通题
- 本地知识库（用于 RAG）：
  - 核心技术栈知识点
  - 高频考点与误区
  - 优秀回答范式（STAR、问题拆解、性能/安全/可维护性表达模板）

### 4.2 多模态模拟面试
- 输入模式：
  - 文字输入
  - 语音输入（ASR 实时/准实时转写）
- 输出模式：
  - 文本输出
  - 语音播报（TTS）
- 多轮会话能力：
  - 根据关键词追问（细节核验、边界场景、取舍分析）
  - 节奏控制（开场-技术主问-压力追问-反问环节-总结）
  - 面试时长与轮次可配置

### 4.3 多维度评估与报告
- 内容分析：
  - 技术正确性
  - 知识深度
  - 逻辑严谨性
  - 与岗位匹配度
- 表达分析：
  - 语速、停顿、清晰度
  - 情绪稳定性/自信度（基于语音特征的启发式指标）
- 报告生成：
  - 总分与分项得分
  - 亮点与不足
  - 可执行改进建议（下一轮训练任务）

### 4.4 个性化提升闭环
- 学习建议推荐：
  - 针对薄弱维度推荐知识点与练习题
  - 推荐表达训练任务（结构化表达、限时回答、反问训练）
- 成长曲线：
  - 按岗位、按维度展示趋势
  - 支持与历史最佳成绩对比
- 历史复盘：
  - 面试记录回放
  - 关键失分片段定位与复训

### 4.5 在线代码编写与运行环境
- 题目展示：
  - 在面试回合中可展示编程题题干、输入输出示例、约束条件与评估要点
- 代码编辑与运行：
  - 提供嵌入式代码编辑器与“运行代码”按钮
  - 至少支持 3 种语言：Python、C++、Java
  - 提供标准输入、运行结果、编译错误、超时/内存超限提示
- 运行安全与隔离：
  - 运行任务需设置 CPU/内存/时长限制
  - 默认禁用外网访问与高风险系统调用

### 4.6 用户登录认证与个性化存储
- 认证能力：
  - 支持用户注册、登录、退出、令牌续期
  - 接口需进行鉴权，确保用户仅访问本人数据
- 个性化记录管理：
  - 为每个用户保存独立面试记录、代码提交记录、评估报告
  - 支持按岗位/时间/难度筛选、查看详情、删除记录

### 4.7 面试强度与维度可配置
- 难度级别：
  - 初级、中级、高级三档题目难度
  - 难度影响题目复杂度、追问深度、评分阈值
- 面试维度：
  - 算法、系统设计、行为面试三类维度可单选/多选
  - 支持按目标岗位配置默认维度权重
- 压力等级：
  - 轻松、适中、紧张三档环境
  - 压力等级影响出题节奏、追问频率、回答时限提示策略

### 4.8 多 Agent 结果评估与证据链
- 评估机制：
  - 面试结果由多个评估 Agent 并行评分（技术正确性、表达清晰度、工程思维、岗位匹配等）
  - 每个 Agent 输出：分项分数、判定理由、证据片段引用
- 综合结论：
  - 使用加权聚合策略生成综合得分与评级
  - 提供 Agent 间分歧分析与最终仲裁逻辑
- 证据链要求：
  - 记录“输入片段 -> Agent 判定 -> 汇总规则 -> 最终结论”的可追溯链路
  - 支持在报告中按问题维度展开查看证据来源

### 4.9 过程计时提示
- 回答用时建议：
  - 每题提供建议作答时长与倒计时提醒
  - 超时后给予分级提示（温和提示/强提醒）
- 时间统计：
  - 统计每题实际用时、每类题型平均用时、与历史均值偏差
  - 在报告中展示“时间管理能力”评分与改进建议

### 4.10 成长预测与能力差距预警
- 通过概率区间预测：
  - 基于历史面试序列（分数、维度表现、时间管理趋势）预测通过概率区间
  - 输出近短期（如 2-4 周）与中期（如 1-2 月）趋势判断
- 能力差距清单：
  - 给出“距目标岗位仍缺失能力”预警项（知识点、表达、工程实践）
  - 每项预警提供优先级、影响说明与建议训练任务

## 5. 大模型与多模态模型选型（满足“本地优先 + 免费额度优先”）

### 5.1 ASR（优先开源本地部署）
- 主方案：`faster-whisper`（`/systran/faster-whisper`）
  - 优势：支持 CPU/GPU、量化推理（INT8/FP16）、部署成熟
  - 用法：本地容器化部署，后端通过 Python 服务调用
- 备选：`whisper.cpp`（纯 C/C++，低资源环境友好）

### 5.2 TTS（优先开源本地部署）
- 主方案：`Coqui TTS XTTS v2`（`/coqui-ai/tts`）
  - 优势：多语言、支持克隆音色、可本地部署
  - 用法：独立 TTS 服务容器，提供 HTTP/gRPC 接口给后端
- 备选：ChatTTS（中文表现较好，可作为中文场景补充）

### 5.3 LLM（优先选择有 API 免费额度供应商）
- 主方案：Google Gemini API（`/websites/ai_google_dev_gemini-api`）
  - 建议模型：`gemini-2.5-flash`（日常面试对话与反馈）
  - 复杂评估可切换：`gemini-2.5-pro`
  - 选择理由：官方提供 Free Tier（按模型不同限制不同），适合比赛阶段快速落地
  - 截至 2026-03-10 的官方配额示例（Free Tier）：
    - `gemini-2.5-flash`：10 RPM / 250,000 TPM / 250 RPD
    - `gemini-2.5-pro`：5 RPM / 250,000 TPM / 100 RPD
- 备选方案（工程降级）：
  - 本地部署开源推理模型（如 Qwen 系列）承担非核心任务，降低 API 依赖

- 第 7 周：成长预测模型与能力差距预警、历史记录管理、可视化报告增强
- 第 8 周：优化与压测、评分标定、容器化部署完善、材料产出与答辩演练
