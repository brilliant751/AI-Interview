# AI 模拟面试项目任务编排（TASK）

## P0 验收规范（强制）
- 所有 `P0` 条目必须同时满足以下 3 个 DoD 字段：
  - 验收场景：明确输入、处理路径、输出与异常路径
  - 量化指标：明确 SLA/错误率/覆盖率/资源上限等可度量指标
  - 自动化验证命令：提供可在 CI 复现的命令
- 下文每个模块均给出适用于本模块全部 `P0` 条目的 DoD 基线；若单条任务有额外约束，以条目内说明为准。

## Objective 1：完成可演示的 MVP 闭环（登录 -> 岗位化面试 -> 代码题运行 -> 评估报告）

### KR1：完成前后端基础架构与统一开发环境，核心链路可联调

#### 模块：文档与工程基建
- [ ] `P0` 输出《系统总体架构说明》：明确前端、后端微服务、数据层、模型网关交互边界
- [ ] `P0` 输出《本地开发与部署指南》：Docker Compose 一键启动与依赖说明
- [ ] `P1` 输出《服务拆分与接口约定文档》：定义 `interview-service`、`auth-service`、`rag-service` 最小职责
- [ ] `P2` 输出《命名规范与分支协作约定》：统一提交规范、目录职责与评审清单
- `P0 DoD（本模块）`
  - 验收场景：新成员按文档在全新环境完成一次本地启动并能访问健康检查接口
  - 量化指标：文档步骤一次通过率 >= 95%；关键依赖缺失率 <= 5%
  - 自动化验证命令：`docker compose config && docker compose up -d && curl -f http://localhost:8000/health`

#### 模块：前端架构
- [ ] `P0` 搭建 React + TypeScript + Vite 项目骨架与路由体系（登录、首页、面试页、报告页）
- [ ] `P0` 建立 Zustand 全局状态（用户态、面试会话态、计时状态）
- [ ] `P1` 封装 Axios API Client（鉴权头、统一错误处理、刷新令牌）
- [ ] `P2` 建立前端埋点规范（页面访问、关键按钮点击、接口耗时）
- `P0 DoD（本模块）`
  - 验收场景：完成登录到面试页再到报告页的路由跳转，状态刷新后可恢复必要会话信息
  - 量化指标：首屏加载 <= 3s（本地）；路由切换错误率 = 0；关键状态丢失率 <= 1%
  - 自动化验证命令：`pnpm lint && pnpm test && pnpm run build`

#### 模块：后端架构
- [ ] `P0` 搭建 FastAPI 网关与基础微服务目录结构（含健康检查、配置加载）
- [ ] `P0` 实现服务间调用规范（请求超时、重试、错误码映射）
- [ ] `P1` 建立异步任务基础能力（用于代码运行轮询与分析任务）
- `P0 DoD（本模块）`
  - 验收场景：网关调用下游服务在正常、超时、5xx 三种场景下返回统一错误结构
  - 量化指标：同步调用超时上限 <= 3s；重试次数 <= 2；错误码映射覆盖率 = 100%
  - 自动化验证命令：`pytest tests/backend/test_gateway*.py -q`

#### 模块：数据与中间件
- [ ] `P0` 完成 PostgreSQL + pgvector + Redis 的容器化编排与连通性验证
- [ ] `P0` 设计并落库 MVP 核心表（用户、岗位、题目、面试会话、回答记录、报告）
- [ ] `P1` 建立基础索引与唯一约束（用户账号、会话查询、岗位题目检索）
- [ ] `P2` 增加冷热数据归档策略设计（历史会话与报告归档规则）
- `P0 DoD（本模块）`
  - 验收场景：完成迁移后可写入并查询 MVP 全链路数据，Redis 可读写会话缓存
  - 量化指标：迁移成功率 = 100%；核心查询 P95 <= 200ms（本地基线）
  - 自动化验证命令：`alembic upgrade head && pytest tests/db/test_schema*.py -q`

### KR2：完成核心业务功能最小集，支持一次完整模拟面试

#### 模块：认证与账号
- [ ] `P0` 实现注册/登录/登出/令牌续期接口，并接入前端登录态
- [ ] `P0` 完成鉴权中间件，确保用户仅可访问本人面试数据
- [ ] `P1` 补充账号安全策略（密码强度、登录失败限流）
- `P0 DoD（本模块）`
  - 验收场景：用户 A、B 分别登录后访问面试记录，跨用户访问被拒绝
  - 量化指标：鉴权误放行率 = 0；令牌续期成功率 >= 99%
  - 自动化验证命令：`pytest tests/auth/test_auth_api*.py -q`

#### 模块：岗位化题库与知识库
- [ ] `P0` 完成岗位管理与题库导入（Java 后端、Web 前端两个岗位）
- [ ] `P0` 实现按岗位与难度抽题接口（支持技术/项目/场景/行为四类）
- [ ] `P1` 建立知识库入库与向量化流程，支持基础语义检索
- `P0 DoD（本模块）`
  - 验收场景：两类岗位各导入 >= 150 题，抽题接口按难度与题型返回合法题目
  - 量化指标：题库入库成功率 >= 99%；抽题接口 4xx/5xx 错误率 <= 1%
  - 自动化验证命令：`pytest tests/question_bank/test_import_and_draw*.py -q`

#### 模块：多轮面试会话（首通）
- [ ] `P0` 定义会话状态机（`INIT`/`OPENING`/`MAIN_QA`/`FOLLOW_UP`/`CANDIDATE_ASK`/`SUMMARY`/`END`）与状态流转规则
- [ ] `P0` 实现开场阶段编排：生成岗位化开场白、确认难度/维度/压力等级参数
- [ ] `P0` 实现主问阶段编排：按岗位题型与难度顺序出题，并维护当前轮次上下文
- [ ] `P0` 实现追问阶段编排：基于回答关键词、缺失点、矛盾点生成追问任务
- [ ] `P0` 实现总结阶段编排：输出本轮面试总结与下一步建议入口
- [ ] `P0` 接入 LLM 网关基础能力：统一提示词模板、超时控制、重试策略、模型降级
- [ ] `P0` 打通 ASR 单步链路：语音输入 -> 转写文本 -> 会话上下文写入
- [ ] `P0` 打通 LLM 单步链路：用户输入 -> 问题生成/追问生成 -> 结构化响应
- [ ] `P0` 打通 TTS 单步链路：LLM 输出文本 -> 语音合成 -> 前端可播放音频
- [ ] `P0` 完成文本模式端到端联调：输入文本 -> LLM 生成问题 -> 前端展示 -> 记录落库
- [ ] `P0` 完成语音模式端到端联调：语音输入 -> ASR -> LLM -> TTS -> 播报与记录落库
- [ ] `P0` 实现轮次控制与结束条件：最大轮次、最大时长、主动结束、异常中止
- [ ] `P1` 实现会话持久化：题目、回答、追问链路、用时、模型调用元数据全量记录
- [ ] `P1` 增加失败容错与回退：ASR/LLM/TTS 任一步失败时自动切换文本模式并给出提示
- [ ] `P2` 增加会话质量监控指标：单轮耗时、模型失败率、追问命中率、语音链路时延
- `P0 DoD（本模块）`
  - 验收场景：完成“文本首通”和“语音首通”两条链路各 1 次，含正常结束与异常中止
  - 量化指标：单轮会话成功率 >= 95%；会话状态机非法流转率 = 0；语音首通链路 P95 <= 6s
  - 自动化验证命令：`pytest tests/interview/test_session_flow*.py -q`

#### 模块：在线代码运行
- [ ] `P0` 接入 Monaco Editor，支持题干展示、代码编辑、提交执行
- [ ] `P0` 打通代码执行服务，支持 Python/C++/Java 三语言基础运行
- [ ] `P0` 返回并展示编译错误、运行结果、超时与内存超限信息
- [ ] `P1` 增加代码运行沙箱限制（CPU/内存/时长/网络隔离）
- [ ] `P2` 增加常见错误模板提示（语法错误/空输入/边界用例遗漏）
- `P0 DoD（本模块）`
  - 验收场景：三语言各提交 1 份正常代码与 1 份异常代码，前端可正确展示执行反馈
  - 量化指标：代码执行成功回传率 >= 99%；结果展示字段完整率 = 100%
  - 自动化验证命令：`pytest tests/code_runner/test_execute_api*.py -q`

#### 模块：评估与报告
- [ ] `P0` 生成基础评估报告（总分、分维度得分、亮点与不足）
- [ ] `P0` 完成报告页可视化展示（分项分数、问题级反馈）
- [ ] `P1` 输出可执行改进建议（下轮训练任务）
- `P0 DoD（本模块）`
  - 验收场景：一次面试结束后自动生成报告并在报告页可查看分项反馈
  - 量化指标：报告生成成功率 >= 99%；报告生成时延 P95 <= 10s
  - 自动化验证命令：`pytest tests/report/test_report_generation*.py -q`
