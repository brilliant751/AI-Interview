# AI模拟面试与能力提升软件-项目计划书

## 1. 项目目标与范围
- 项目名称：面向计算机相关专业学生的 AI 模拟面试与能力提升平台
- 目标用户：高校计算机相关专业学生（重点覆盖毕业求职阶段）
- 建设目标：
  - 提供岗位化、多轮、多模态模拟面试能力
  - 提供可量化、可追踪、可解释的能力评估报告
  - 形成“练习-评估-提升-复盘”的闭环学习路径
- 首期岗位范围（至少 2 个，满足赛题要求）：
  - Java 后端开发工程师
  - Web 前端开发工程师

## 2. 具体技术栈选型

### 2.1 前端
- React 18
- TypeScript 7.0（说明：若实际工程环境暂不可用该版本，则先采用当前稳定版并保持类型规范一致）
- Vite
- React Router
- Zustand
- Axios
- 关键前端能力：
  - 面试对话页（文本/语音输入）
  - 实时状态管理（会话状态、计时、问题轮次、得分草稿）
  - 评估报告可视化（雷达图、趋势图、题型分布）
  - 历史记录与岗位切换

### 2.2 后端
- Python 3.10
- FastAPI
- 模块划分：
  - `interview-service`：会话编排、追问策略、评分汇总
  - `rag-service`：题库检索、知识库检索、上下文构建
  - `analysis-service`：内容分析、表达分析、报告生成
  - `model-gateway`：统一封装 LLM/ASR/TTS 调用与降级

### 2.3 数据层
- PostgreSQL：业务主库（用户、岗位、题库、会话、评估报告）
- pgvector：向量检索（知识片段、题目语义索引、优秀回答示例）
- Redis：会话缓存、限流、短期上下文、异步任务状态

### 2.4 部署
- Docker + Docker Compose
- 容器建议：
  - `frontend`、`backend-api`、`worker`
  - `postgres`（启用 pgvector）
  - `redis`
  - `asr-local`（本地语音识别服务）
  - `tts-local`（本地语音合成服务）

## 3. 系统总体架构（简版）
- 前端负责交互与可视化，后端统一编排对话与评估流程
- RAG 从“岗位题库 + 本地知识库”召回上下文，提供给 LLM 进行追问和打分解释
- ASR 将语音转文本；TTS 负责 AI 面试官语音播报
- 面试完成后由分析模块生成结构化报告并写入数据库

## 4. 需要完成的功能点

### 4.1 岗位化题库与知识库
- 岗位配置管理：
  - 支持岗位创建、修改、启停
  - 每岗位独立评分维度权重（技术深度/表达逻辑/岗位匹配等）
- 题库建设（每岗位不少于 150 题，覆盖四类）：
  - 技术知识题
  - 项目经历深挖题
  - 场景设计题
  - 行为与沟通题
- 本地知识库（用于 RAG）：
  - 核心技术栈知识点
  - 高频考点与误区
  - 优秀回答范式（STAR、问题拆解、性能/安全/可维护性表达模板）

### 4.2 多模态模拟面试
- 输入模式：
  - 文字输入
  - 语音输入（ASR 实时/准实时转写）
- 输出模式：
  - 文本输出
  - 语音播报（TTS）
- 多轮会话能力：
  - 根据关键词追问（细节核验、边界场景、取舍分析）
  - 节奏控制（开场-技术主问-压力追问-反问环节-总结）
  - 面试时长与轮次可配置

### 4.3 多维度评估与报告
- 内容分析：
  - 技术正确性
  - 知识深度
  - 逻辑严谨性
  - 与岗位匹配度
- 表达分析：
  - 语速、停顿、清晰度
  - 情绪稳定性/自信度（基于语音特征的启发式指标）
- 报告生成：
  - 总分与分项得分
  - 亮点与不足
  - 可执行改进建议（下一轮训练任务）

### 4.4 个性化提升闭环
- 学习建议推荐：
  - 针对薄弱维度推荐知识点与练习题
  - 推荐表达训练任务（结构化表达、限时回答、反问训练）
- 成长曲线：
  - 按岗位、按维度展示趋势
  - 支持与历史最佳成绩对比
- 历史复盘：
  - 面试记录回放
  - 关键失分片段定位与复训

## 5. 大模型与多模态模型选型（满足“本地优先 + 免费额度优先”）

### 5.1 ASR（优先开源本地部署）
- 主方案：`faster-whisper`（`/systran/faster-whisper`）
  - 优势：支持 CPU/GPU、量化推理（INT8/FP16）、部署成熟
  - 用法：本地容器化部署，后端通过 Python 服务调用
- 备选：`whisper.cpp`（纯 C/C++，低资源环境友好）

### 5.2 TTS（优先开源本地部署）
- 主方案：`Coqui TTS XTTS v2`（`/coqui-ai/tts`）
  - 优势：多语言、支持克隆音色、可本地部署
  - 用法：独立 TTS 服务容器，提供 HTTP/gRPC 接口给后端
- 备选：ChatTTS（中文表现较好，可作为中文场景补充）

### 5.3 LLM（优先选择有 API 免费额度供应商）
- 主方案：Google Gemini API（`/websites/ai_google_dev_gemini-api`）
  - 建议模型：`gemini-2.5-flash`（日常面试对话与反馈）
  - 复杂评估可切换：`gemini-2.5-pro`
  - 选择理由：官方提供 Free Tier（按模型不同限制不同），适合比赛阶段快速落地
- 备选方案（工程降级）：
  - 本地部署开源推理模型（如 Qwen 系列）承担非核心任务，降低 API 依赖

## 6. 里程碑计划（建议 8 周）
- 第 1-2 周：需求拆解、数据结构设计、题库模板与评分标准确定
- 第 3-4 周：前后端骨架、用户流程、岗位化题库/知识库导入、RAG 基础链路
- 第 5 周：ASR/TTS 接入、语音链路打通、多轮追问策略实现
- 第 6 周：多维度评分与报告生成、历史记录与可视化
- 第 7 周：优化与压测、提示词与评分标定、容器化部署完善
- 第 8 周：材料产出（PPT、详细方案、演示视频、知识库打包）与答辩演练
