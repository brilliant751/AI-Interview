# 系统架构说明

更新时间：2026-06-28

本文说明 AI-Interview 的系统分层、核心模块、数据流、异步任务和降级机制。

## 1. 总体架构

AI-Interview 是前后端分离应用：

- 前端：React + Vite + TypeScript + Ant Design。
- 后端：FastAPI + SQLite + 本地材料/向量数据。
- 模型与语音：通过 provider 封装 OpenAI 兼容接口、Ollama、FunASR、PaddleSpeech 和 mock。
- 数据处理：通过脚本完成材料校验、归一化、题库构建、向量库构建和评估。
- 测试与 CI：pytest、ruff、Vitest、ESLint、Playwright、GitHub Actions。

运行时请求路径：

```text
Browser
  -> React Router 页面
  -> TanStack Query / axios client
  -> FastAPI /api/v1/*
  -> service 层
  -> repository / provider / data scripts
  -> SQLite / material files / vector data
```

## 2. 前端架构

前端代码位于 `frontend/src/`：

- `api/`：HTTP client 和业务 API 封装。
- `components/`：布局、欢迎页、provider 状态等公共组件。
- `hooks/`：倒计时等可复用 hook。
- `pages/`：页面级组件。
- `router/`：路由配置与权限保护。
- `stores/`：Zustand 状态。
- `styles/`：全局样式。
- `utils/`：日历等工具函数。
- `monaco/`：编程练习编辑器配置。

前端请求统一通过 `apiClient` 发起：

- 默认 baseURL：`import.meta.env.VITE_API_BASE ?? http://localhost:18500/api/v1`
- 自动注入 Bearer token。
- 自动注入 `X-Idempotency-Key`。
- 401 时尝试 refresh token 并重放原请求。
- 统一解析后端错误结构。

路由分为：

- 公开路由：登录、注册、找回密码、重置密码、欢迎页。
- 登录保护路由：首页、简历、岗位、面试、预约、报告、练习等。
- 管理员路由：知识库重建、题库管理。

## 3. 后端架构

后端代码位于 `backend/app/`：

- `main.py`：FastAPI 应用创建、CORS、中间件、生命周期初始化。
- `api/v1/`：接口路由。
- `core/`：配置、错误、鉴权、安全、日志。
- `models/`：Pydantic 请求与响应模型。
- `repositories/`：SQLite 数据访问层。
- `services/`：业务服务、模型 provider、报告 worker、轮次 worker。

应用启动时会初始化：

- SQLite repository。
- 数据库 schema。
- 编程题同步。
- 材料导入服务。
- 认证服务。
- 面试服务。
- 报告 worker。
- 轮次 worker。
- 练习服务。
- 编程练习服务。

## 4. 业务域划分

### 4.1 认证域

负责注册、登录、刷新、退出、找回密码、重置密码、当前用户查询。密码采用 PBKDF2 哈希；refresh token 只保存哈希；登录有简易限流。

### 4.2 简历域

负责简历上传、解析文本、列表、原文件读取和删除保护。简历与用户隔离，历史面试引用的简历不会被误删。

### 4.3 JD 与公司域

负责公司列表、系统预置 JD、用户上传 JD、JD 删除和面试会话快照。面试创建时会保存 JD 标题和正文快照，保证后续报告可复现。

### 4.4 面试域

负责创建会话、提交轮次、音频轮次、轮次异步任务、暂停、恢复、开始预约面试、结束面试、状态查询和回放。

### 4.5 预约域

负责单场面试预约、列表、详情、取消、开始、日历文件和外部日历链接。

### 4.6 报告域

负责报告异步生成、报告查询、列表和重试。报告生成优先调用 LLM，失败后回退到规则报告。

### 4.7 题库练习域

负责题库练习会话、答题、结束、概览、记录和管理端题库维护。

### 4.8 编程练习域

负责编程题列表、会话、草稿、运行自测、正式提交、记录查询和本地代码执行。

### 4.9 材料导入域

负责异步执行数据脚本、记录任务状态、幂等、全量任务并发保护和报告路径返回。

## 5. 面试链路数据流

```text
用户选择简历/JD/难度/模式
  -> POST /interviews
  -> InterviewService 创建 session
  -> 返回第一题
  -> 用户文本或音频作答
  -> POST /interviews/{id}/turns 或 /turns/audio
  -> 创建 turn job
  -> TurnWorker 执行 ASR、检索、LLM 生成、TTS
  -> 保存 turn 与 pipeline_meta
  -> 前端轮询 job result
  -> 更新下一题和历史记录
  -> 用户结束面试
  -> ReportWorker 生成报告
```

## 6. Provider 与降级

系统把外部能力封装为 provider：

- LLM：mock、OpenAI 兼容、Ollama。
- ASR：mock、OpenAI、FunASR、PaddleSpeech 兼容。
- TTS：mock、OpenAI、PaddleSpeech。
- Embedding/检索：本地脚本和 Chroma 数据目录。

面试轮次会返回 `pipeline_meta`：

- 输入来源。
- provider 名称。
- provider 状态。
- 降级标记。
- trace ID。
- 延迟。
- 生成模式：`local_ai`、`fallback_template`、`mock`。

当真实 provider 不可用时，系统尽量使用模板和规则回退，保证演示流程可走通。

## 7. 异步任务

系统中有三类典型异步任务：

- 面试轮次任务：提交答案后创建 job，前端轮询结果。
- 报告任务：结束面试后异步生成报告。
- 材料导入任务：管理员触发后异步执行脚本流水线。

异步任务的目标：

- 避免长耗时模型调用阻塞主请求。
- 让前端可以展示处理中状态。
- 在失败时保留错误信息，便于重试和排查。

## 8. 数据存储

主存储是 SQLite，默认路径：

- `backend/assets/data/sqlite/interview.db`

其他数据：

- 材料源：`backend/assets/material/`
- 归一化数据：`backend/assets/data/normalized/`
- 向量库：`backend/assets/data/chroma/`
- 构建报告：`backend/assets/data/reports/`

SQLite 适合本地演示和 MVP。若面向多用户生产环境，应迁移到 PostgreSQL 或其他服务端数据库，并引入独立对象存储和任务队列。

## 9. 安全边界

已实现：

- Bearer token 鉴权。
- refresh token 轮换与撤销。
- 用户数据按 `user_id` 隔离。
- 管理端路由角色限制。
- 写请求幂等键。
- 简历删除占用保护。
- 题库上传路径校验。
- 密码复杂度校验。

仍需注意：

- 本地代码执行不是强沙箱，不适合直接暴露给不可信公网用户。
- SQLite 不适合高并发生产环境。
- 默认 mock/dev 配置不应作为生产安全配置。

## 10. 可观测性

后端中间件会记录 `/api/` 请求日志，包括方法、路径、状态码、延迟和客户端。405 时会额外输出允许的方法。面试链路还会记录 trace ID、provider 状态和降级标记。

建议演示或上线前检查：

- `/api/v1/admin/providers/health`
- 后端控制台日志。
- `backend/assets/data/reports/*.json`
- Playwright 失败 artifact。
