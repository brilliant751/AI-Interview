# 数据与知识库说明

更新时间：2026-06-28

本文说明 AI-Interview 的数据目录、SQLite 表、材料格式、题库构建、知识库重建和备份建议。

## 1. 数据目录

主要数据位于：

```text
backend/assets/data/
```

常见子目录：

- `sqlite/`：SQLite 数据库。
- `normalized/`：归一化后的材料数据。
- `chroma/`：向量库和别名文件。
- `reports/`：材料校验、题库构建、向量库构建、导入校验等报告。

材料源位于：

```text
backend/assets/material/
```

根目录 `data/` 保留开发和演示过程中的数据产物，不作为运行时唯一事实来源。

## 2. SQLite 数据库

默认数据库路径：

```text
backend/assets/data/sqlite/interview.db
```

可通过环境变量覆盖：

```text
AI_INTERVIEW_DB_PATH
```

主要表：

| 表 | 说明 |
| --- | --- |
| `user_accounts` | 用户账号 |
| `auth_refresh_tokens` | 刷新令牌哈希 |
| `auth_password_reset_tokens` | 密码重置令牌哈希 |
| `resumes` | 简历元数据和解析文本 |
| `job_descriptions` | JD |
| `companies` | 公司 |
| `voice_tone_profiles` | 面试官语气配置 |
| `interview_sessions` | 面试会话 |
| `interview_schedules` | 面试预约 |
| `interview_turns` | 面试问答轮次 |
| `interview_turn_jobs` | 轮次异步任务 |
| `interview_reports` | 面试报告 |
| `request_idempotency` | 幂等响应缓存 |
| `practice_choice_questions` | 题库练习题 |
| `practice_sessions` | 题库练习会话 |
| `practice_session_questions` | 练习题目快照 |
| `practice_answers` | 练习答案 |

| `coding_sessions` | 编程练习会话 |

| `coding_submissions` | 编程提交 |

后端启动时会调用 `InterviewRepository.init_schema()` 创建和补齐表结构。

## 4. 材料源目录

固定材料目录：

| 类型 | 路径 |
| --- | --- |
| Java 面试题 | `backend/assets/material/java/java-interview/` |
| Java 知识库 | `backend/assets/material/java/java-knowledge/` |
| Web 面试题 | `backend/assets/material/web/interview.md` |
| Web 知识库 | `backend/assets/material/web/knowledge.md` |
| 编程练习题 | `backend/assets/material/coding/programming_practice_questions.json` |

材料目录是题库管理和知识库构建的事实来源。管理员页面上传或录入题库时，也会写入该目录。

## 5. 题库 Markdown 格式

Java 题库：

```markdown
## 第 1 题：标题

### 题干

题干内容。

### 类别

技术

### 解析

解析内容。
```

Web 题库：

```markdown
### 第 1 题：标题

#### 题干

题干内容。

#### 类别

场景

#### 解析

解析内容。
```

类别只允许：

- 技术。
- 项目。
- 场景。
- 行为。

英文内部值：

- `technical`
- `project`
- `scenario`
- `behavior`

## 6. 编程题 JSON 格式

编程题材料文件：

```text
backend/assets/material/coding/programming_practice_questions.json
```

题目通常包含：

- `question_id`
- `slug`
- `title`
- `difficulty`
- `topic_tags`
- `prompt_markdown`
- `input_spec`
- `output_spec`
- `constraints_text`
- `sample_cases`
- `judge_cases`
- `self_test_case`
- `starter_codes`

后端启动时会同步内置编程题到 SQLite。

## 7. 数据初始化

首次初始化推荐执行：

```bash
python backend/assets/scripts/data/validate_materials.py --strict
python backend/assets/scripts/data/normalize_materials.py
python backend/assets/scripts/data/build_question_bank.py
python backend/assets/scripts/data/import_coding_practice_questions.py
python backend/assets/scripts/data/build_knowledge_vectorstore.py
```

跨平台启动脚本会在数据库不存在时自动初始化数据。

## 8. 数据脚本

常用脚本：

- `validate_materials.py`：校验材料格式。
- `normalize_materials.py`：材料归一化。
- `build_question_bank.py`：构建题库。
- `import_coding_practice_questions.py`：导入编程题。
- `build_knowledge_vectorstore.py`：构建知识向量库。
- `chunk_with_ollama.py`：使用模型进行知识分块预处理。
- `evaluate_retrieval.py`：评估检索效果。

CI 会执行 dry-run：

```bash
python backend/assets/scripts/data/validate_materials.py --strict
python backend/assets/scripts/data/normalize_materials.py --dry-run
python backend/assets/scripts/data/build_question_bank.py --dry-run
python backend/assets/scripts/data/build_knowledge_vectorstore.py --dry-run
```

## 9. 知识库与检索

知识库构建产物位于：

```text
backend/assets/data/chroma/
```

常见配置：

- `AI_INTERVIEW_CHROMA_DIR`
- `AI_INTERVIEW_KB_COLLECTION_ALIAS_FILE`
- `AI_INTERVIEW_RETRIEVAL_FALLBACK_ENABLED`
- `AI_INTERVIEW_EMBEDDING_MODEL`
- `AI_INTERVIEW_EMBED_BATCH_SIZE`

如果 Chroma 或 embedding 不可用，系统可配置回退到 JSONL/模板路径，保证面试流程继续。

## 10. 构建报告

构建报告位于：

```text
backend/assets/data/reports/
```

常见文件：

- `material_validation_report.json`
- `question_bank_build_report.json`
- `knowledge_vectorstore_build_report.json`
- `knowledge_vectorstore_build_report_latest.json`
- `import_verify_report.md`

每次数据构建后建议检查报告，确认通过数量、失败数量和错误明细。

## 11. 备份与恢复

演示或批量导入前建议备份：

```powershell
Copy-Item -Recurse backend/assets/data backend/assets/data.backup
```

恢复时：

1. 停止后端。
2. 备份当前异常数据目录。
3. 用备份目录覆盖 `backend/assets/data`。
4. 重新启动后端。
5. 打开 `/docs` 和核心页面验证。

## 12. 数据维护注意事项

- 不要手动编辑 SQLite，除非已备份。
- 不要把临时文件放进材料目录。
- 上传题库前先用小文件验证格式。
- 全量知识库重建耗时取决于本地模型和 embedding 服务。
- 演示前优先使用 mock provider 保证流程稳定。
- 接入真实 provider 后，应重新检查报告质量和题目生成质量。
