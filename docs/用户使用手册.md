# 用户使用手册

更新时间：2026-06-28

本文面向普通用户，说明 AI-Interview 的主要页面、操作流程和注意事项。

## 1. 登录与账号

打开前端页面：

- 默认地址：`http://localhost:5173`

公开入口包括：

- `/login`：登录账号。
- `/register`：注册账号。
- `/forgot-password`：找回密码。
- `/reset-password`：重置密码。

注册时需要填写邮箱、昵称和密码。密码至少 8 位，并且需要同时包含字母和数字。注册成功后，需要返回登录页登录。登录后系统会保存 access token 与 refresh token；会话过期时前端会自动刷新。

如果登录失败：

- 检查邮箱格式和密码。
- 如果连续登录失败过多，后端会触发登录限流，需要稍后重试。
- 如果页面提示 `Network Error`，优先检查前端 `VITE_API_BASE` 是否指向正在运行的后端端口。

## 2. 首页概览

登录后进入 `/overview`。首页用于快速进入训练任务并查看状态：

- 开始新面试。
- 预约面试。
- 继续上次面试。
- 查看我的报告。
- 查看最近预约日历。
- 查看系统 provider 状态。

顶部右侧会显示用户菜单和当天预约提醒。当天有待进行或进行中的面试时，系统会弹出通知并引导进入面试大厅。

## 3. 简历管理

进入 `/resumes` 管理简历。

支持操作：

- 上传 PDF、DOC、DOCX 简历。
- 查看简历列表。
- 查看解析状态。
- 预览 PDF 简历。
- 下载非 PDF 简历。
- 删除未被占用的简历。

注意事项：

- 创建 AI 面试前必须选择一份简历。
- 已绑定进行中面试的简历不能删除。
- 简历解析状态为 `READY` 时最适合用于面试追问和报告生成。

## 4. 岗位库

进入 `/jobs` 管理岗位描述 JD。

系统提供两类岗位描述：

- 系统预置：可查看，不可删除。
- 我的上传：用户录入，可查看和删除。

用户可以：

- 按岗位方向筛选，例如后端开发、前端开发、算法、产品等。
- 输入自定义岗位方向。
- 按岗位标题搜索。
- 按公司筛选。
- 新增岗位描述，填写标题、公司和岗位正文。
- 点击卡片查看完整 JD。

面试时可以绑定 JD。绑定后，AI 问题生成和报告会参考岗位描述内容。

## 5. AI 面试

进入 `/interview` 开始或恢复面试。

### 5.1 创建面试

创建面试需要选择或填写：

- 简历。
- 岗位方向或 JD。
- 难度：简单、中等、困难。
- 输入方式：文本或语音。
- 输出方式：文本或语音。
- 问题类型：项目、技术、场景。
- 面试名称。
- 面试官语气。

创建成功后，系统进入答题页并展示第一题。

### 5.2 文本面试

文本模式下，问题区展示当前问题，回答区提供文本框。用户输入答案后点击提交。系统会异步生成下一题，并更新历史记录。

注意事项：

- 文本作答有倒计时提示。
- 回答应尽量包含背景、行动、结果和反思，报告质量会更高。

### 5.3 语音面试

语音输入模式下，浏览器会请求麦克风权限。用户可以选择麦克风设备。题目播放后，系统进入思考倒计时；用户可以等待倒计时自动录音，也可以手动开始。

语音作答规则：

- 录音最长 3 分钟。
- 达到上限会自动停止并提交。
- 题目语音播放中不能录音。
- 如果浏览器拦截自动播放，可手动重播题目。

语音输出模式下，系统会返回可播放的题目音频。如果 provider 不可用，页面会展示降级提示，并保留文本题目。

### 5.4 暂停、继续与结束

面试过程中可以：

- 暂停并保存进度。
- 通过 `/interview/{interview_id}` 继续面试。
- 结束面试并触发报告生成。

结束后，系统会跳转报告详情页。报告可能先显示 `GENERATING`，稍后自动刷新为 `READY`。

## 6. 面试预约

进入 `/schedules` 创建和管理预约。

预约字段：

- 标题。
- 开始时间。
- 时长：20、45、60 分钟。
- 简历。
- 岗位方向或 JD。
- 难度。
- 输入/输出模式。
- 问题类型。
- 面试官语气。

预约列表支持：

- 按日期查看。
- 按状态查看。
- 查看详情。
- 取消预约。
- 到点后开始面试。
- 下载 `.ics` 日历文件。
- 打开 Google Calendar 或 Outlook Calendar 链接。

预约状态：

- `scheduled`：已预约。
- `ready`：可开始。
- `in_progress`：进行中。
- `completed`：已完成。
- `missed`：已错过。
- `cancelled`：已取消。

## 7. 面试记录与回放

进入 `/history` 查看历史面试。列表包括：

- 面试名称。
- 简历。
- 岗位。
- 难度。
- 状态。
- JD。
- 开始与结束时间。
- 问答轮次数。
- 总分。

点击记录可进入 `/history/{interview_id}` 查看回放。回放页展示每轮面试官问题和用户回答，适合复盘表达结构和关键遗漏。

## 8. 我的报告

进入 `/report` 查看报告列表，进入 `/report/{interview_id}` 查看报告详情。

报告状态：

- `GENERATING`：生成中。
- `READY`：已完成。
- `FAILED`：生成失败。

报告详情包括：

- 总分。
- 优势。
- 待改进项。
- 建议。
- 录用建议。
- 维度评分。
- 能力雷达图。
- JD、简历、回答三方对齐。
- 关键问题深度分析。
- 风险清单。

如果报告生成失败，可以点击重试生成。

## 9. 题库练习

进入 `/practice` 开始题库练习。

可配置：

- 岗位域：Java 或 Web。
- 模式：顺序练习或追问占位模式。
- 题量。
- 类别：技术、项目、场景、行为。

练习中按题作答。完成后可查看练习记录和单场明细。

## 10. 编程练习

进入 `/coding-practice` 查看编程题列表。

支持语言：

- C++11。
- Java。
- JavaScript。

进入题目后可以：

- 查看题面、输入输出说明、约束、样例。
- 编写代码。
- 运行自测。
- 正式提交判题。
- 查看编译错误、运行错误、超时、错误答案或通过结果。

注意事项：

- 后端在本地临时目录中执行代码。
- 适合本地演示、课程训练和小范围使用。
- 面向公网使用时需要额外引入严格沙箱。

## 11. 常见问题

### 注册时报 Network Error

通常是前端请求端口和后端启动端口不一致。确认：

- 后端是否运行。
- 前端 `VITE_API_BASE` 是否指向正确地址。
- 如果后端在 `8000`，前端启动前设置 `VITE_API_BASE=http://localhost:8000/api/v1`。

### 面试问题没有真实 AI 感

检查 provider 配置。默认 mock 模式会使用模板问题。需要配置 OpenAI 兼容接口或 Ollama 才会调用真实 LLM。

### 语音不可用

检查浏览器麦克风权限、ASR/TTS provider 配置和 `/api/v1/admin/providers/health`。

### 编程练习无法运行

检查本机是否安装 Node.js、`g++`、`javac`/`java`，以及它们是否在 PATH 中。
