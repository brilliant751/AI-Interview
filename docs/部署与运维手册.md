# 部署与运维手册

更新时间：2026-06-28

本文说明 AI-Interview 的本地启动、环境变量、provider 配置、端口、备份恢复和常见故障处理。

## 1. 环境要求

基础依赖：

- Python 3.11+
- Node.js 18+
- npm 9+

编程练习依赖：

- `g++`
- `javac`
- `java`
- `node`

可选模型能力：

- Ollama。
- FunASR。
- PaddleSpeech。
- OpenAI 兼容服务。

## 2. 推荐启动方式

Windows PowerShell：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\start-windows.ps1
```

Windows CMD：

```cmd
scripts\start-windows.cmd
```

macOS/Linux：

```bash
bash scripts/start-unix.sh
```

默认地址：

- 前端：`http://localhost:5173`
- 后端：`http://localhost:18500`
- Swagger：`http://localhost:18500/docs`

## 3. 单独启动后端

在仓库根目录：

```bash
PYTHONPATH=backend python -m uvicorn app.main:app --host 0.0.0.0 --port 8000
```

Windows PowerShell 示例：

```powershell
$env:PYTHONPATH="backend"
python -m uvicorn app.main:app --host 0.0.0.0 --port 8000
```

## 4. 单独启动前端

进入 `frontend`：

```bash
npm run dev
```

如果后端运行在 `8000`：

```powershell
$env:VITE_API_BASE="http://localhost:8000/api/v1"
npm run dev
```

如果未设置 `VITE_API_BASE`，前端默认请求 `http://localhost:18500/api/v1`。

## 5. 常用环境变量

启动脚本变量：

| 变量 | 默认值 | 说明 |
| --- | --- | --- |
| `BACKEND_HOST` | `0.0.0.0` | 后端监听地址 |
| `BACKEND_PORT` | `18500` | 后端端口 |
| `FRONTEND_HOST` | `0.0.0.0` | 前端监听地址 |
| `FRONTEND_PORT` | `5173` | 前端端口 |
| `START_FRONTEND` | `1` | 是否启动前端 |
| `BACKEND_RELOAD` | `0` | 是否启用 uvicorn reload |
| `SKIP_INSTALL` | `0` | 是否跳过依赖安装 |
| `SKIP_DATA_INIT` | `0` | 是否跳过数据初始化 |
| `VITE_API_BASE` | `http://localhost:${BACKEND_PORT}/api/v1` | 前端 API 地址 |

后端应用变量：

| 变量 | 默认值 | 说明 |
| --- | --- | --- |
| `AI_INTERVIEW_APP_ENV` | `dev` | 应用环境 |
| `AI_INTERVIEW_DB_PATH` | `backend/assets/data/sqlite/interview.db` | SQLite 路径 |
| `AI_INTERVIEW_CHROMA_DIR` | `backend/assets/data/chroma` | Chroma 数据目录 |
| `AI_INTERVIEW_AUTH_ENABLE_DEV_STATIC_TOKEN` | `false` | 是否启用开发静态 token |
| `AI_INTERVIEW_RETRIEVAL_FALLBACK_ENABLED` | `false` | 是否启用检索回退 |
| `AI_INTERVIEW_JWT_SECRET` | `dev-jwt-secret` | JWT 密钥 |
| `AI_INTERVIEW_ACCESS_TOKEN_TTL_MINUTES` | `30` | access token 有效期 |
| `AI_INTERVIEW_REFRESH_TOKEN_TTL_DAYS` | `7` | refresh token 有效期 |

Provider 变量：

| 变量 | 默认值 | 说明 |
| --- | --- | --- |
| `AI_INTERVIEW_LLM_PROVIDER` | `mock` | LLM provider |
| `AI_INTERVIEW_ASR_PROVIDER` | `mock` | ASR provider |
| `AI_INTERVIEW_TTS_PROVIDER` | `mock` | TTS provider |
| `AI_INTERVIEW_OPENAI_API_KEY` | 空 | OpenAI 兼容密钥 |
| `AI_INTERVIEW_PROVIDER_BASE_URL` | 空 | OpenAI 兼容地址 |
| `AI_INTERVIEW_PROVIDER_TRUST_ENV` | `true` | provider 是否读取系统代理 |
| `AI_INTERVIEW_OLLAMA_BASE_URL` | `http://localhost:11434` | Ollama 地址 |
| `AI_INTERVIEW_FUNASR_BASE_URL` | `http://localhost:10095` | FunASR 兼容地址 |
| `AI_INTERVIEW_PADDLESPEECH_BASE_URL` | `http://localhost:8888` | PaddleSpeech 兼容地址 |

## 6. Provider 模式

### 6.1 稳定演示模式

使用 mock：

```powershell
$env:AI_INTERVIEW_LLM_PROVIDER="mock"
$env:AI_INTERVIEW_ASR_PROVIDER="mock"
$env:AI_INTERVIEW_TTS_PROVIDER="mock"
```

优点：不依赖外部服务，适合演示和 CI。

### 6.2 OpenAI 兼容模式

```powershell
$env:AI_INTERVIEW_LLM_PROVIDER="openai"
$env:AI_INTERVIEW_OPENAI_API_KEY="<key>"
$env:AI_INTERVIEW_PROVIDER_BASE_URL="https://open.bigmodel.cn/api/paas/v4/"
```

如果 ASR/TTS 也使用 OpenAI 兼容 provider，设置：

```powershell
$env:AI_INTERVIEW_ASR_PROVIDER="openai"
$env:AI_INTERVIEW_TTS_PROVIDER="openai"
```

### 6.3 本地模型模式

```powershell
$env:AI_INTERVIEW_LLM_PROVIDER="ollama"
$env:AI_INTERVIEW_ASR_PROVIDER="funasr"
$env:AI_INTERVIEW_TTS_PROVIDER="paddlespeech"
```

本地模式对机器环境要求更高。启动前先确认对应服务、模型和 Python 包可用。

## 7. 端口排查

Windows 查看端口：

```powershell
Get-NetTCPConnection -State Listen | Where-Object { $_.LocalPort -in 18500,5173,8000 }
```

查看进程：

```powershell
Get-CimInstance Win32_Process -Filter "ProcessId=<PID>" | Select-Object ProcessId,CommandLine
```

如果端口冲突，可设置：

```powershell
$env:BACKEND_PORT="18501"
$env:FRONTEND_PORT="5174"
.\scripts\start-windows.ps1
```

## 10. 备份与恢复

需要备份：

- `backend/assets/data/sqlite/interview.db`
- `backend/assets/data/chroma/`
- `backend/assets/data/reports/`
- `backend/assets/material/`

PowerShell 备份示例：

```powershell
$stamp = Get-Date -Format "yyyyMMdd-HHmmss"
Copy-Item -Recurse backend/assets/data "backend/assets/data.backup-$stamp"
Copy-Item -Recurse backend/assets/material "backend/assets/material.backup-$stamp"
```

恢复流程：

1. 停止前后端。
2. 备份当前异常目录。
3. 用备份覆盖数据目录。
4. 启动后端。
5. 验证 `/docs`、登录、简历列表、题库练习、面试创建。

## 11. 常见故障

### 前端 Network Error

原因通常是后端端口不一致。检查：

- 后端是否监听。
- `VITE_API_BASE` 是否匹配。
- 浏览器控制台请求 URL。

### 注册成功但登录失败

检查密码是否正确、账号是否 active、登录限流是否触发。

### 语音播放或录音失败

检查：

- 浏览器麦克风权限。
- HTTPS/localhost 环境。
- provider 健康。
- ASR/TTS 配置。

### 报告一直生成中

检查：

- 面试是否已结束。
- 后端 report worker 是否运行。
- `interview_reports` 记录。
- 后端日志是否有 LLM 错误。

### 编程练习编译失败

检查：

- `g++` 是否可用。
- `javac`/`java` 是否可用。
- Java 版本是否支持 `--release 21`，不支持时系统会自动降级普通 `javac`。

### 知识库重建失败

检查：

- 材料格式。
- Ollama 是否可用。
- Chroma 目录权限。
- embedding 模型名称是否匹配。
- `backend/assets/data/reports/` 中的报告。

## 12. 生产化建议

当前项目更适合本地演示、课程项目和 MVP 验证。生产化前建议：

- SQLite 迁移到 PostgreSQL。
- 文件存储迁移到对象存储。
- 异步任务迁移到队列系统。
- 编程练习执行引入容器沙箱。
- 密钥改为安全的 secret 管理。
- 加入 HTTPS、CSRF 策略和更严格 CORS。
- 日志接入集中式可观测平台。
- 为 provider 调用增加熔断、重试、限流和成本监控。
