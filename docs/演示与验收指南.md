# 演示与验收指南

更新时间：2026-06-28

本文用于课程演示、项目答辩、产品验收和交付检查。

## 1. 演示前准备

### 1.1 环境检查

确认已安装：

- Python 3.11+
- Node.js 18+
- npm 9+
- `g++`
- `javac` / `java`

推荐使用 mock provider，保证演示稳定：

```powershell
$env:AI_INTERVIEW_LLM_PROVIDER="mock"
$env:AI_INTERVIEW_ASR_PROVIDER="mock"
$env:AI_INTERVIEW_TTS_PROVIDER="mock"
```

启动：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\start-windows.ps1
```

检查：

- 前端：`http://localhost:5173`
- 后端：`http://localhost:18500/docs`

### 1.2 数据检查

确认：

- 有可用简历或准备一个 PDF 简历。
- 有 Java/Web 题库。
- 有编程题。
- 管理员账号可登录。
- provider 健康接口可返回。

### 1.3 浏览器准备

建议：

- 使用 Chrome 或 Edge。
- 打开开发者工具备用。
- 允许 localhost 麦克风权限。
- 关闭会干扰录音的其他软件。

## 2. 推荐演示路线

### 路线 A：完整用户闭环

用时：10 到 15 分钟。

1. 打开欢迎页。
2. 注册账号。
3. 登录进入首页。
4. 上传简历。
5. 新增岗位描述。
6. 创建 AI 面试。
7. 完成一轮文本回答。
8. 暂停面试。
9. 从会话页恢复。
10. 再提交一轮回答。
11. 结束面试。
12. 查看报告生成状态。
13. 打开历史记录和回放。

重点讲解：

- 简历、JD、知识库共同驱动问题生成。
- 轮次异步任务避免阻塞。
- 报告提供维度评分和证据。
- 暂停/恢复适合长流程训练。

### 路线 B：预约与日历

用时：5 到 8 分钟。

1. 进入面试预约页。
2. 创建一场 20 或 45 分钟预约。
3. 查看日历视图。
4. 打开预约详情。
5. 下载 `.ics`。
6. 展示 Google/Outlook 链接。
7. 如果时间可开始，点击开始面试。

重点讲解：

- 预约状态流转。
- 日历导出。
- 首页当天预约提醒。

### 路线 C：练习能力

用时：8 到 10 分钟。

1. 进入题库练习。
2. 选择 Java、技术类别、题量。
3. 提交一道题。
4. 查看练习记录。
5. 进入编程练习。
6. 选择题目。
7. 切换语言。
8. 运行自测。
9. 正式提交。

重点讲解：

- 题库练习适合面试问答专项训练。
- 编程练习提供本地判题能力。
- 题解代码不返回给前端。

### 路线 D：管理员后台

用时：8 到 12 分钟。

1. 使用管理员账号登录。
2. 进入 provider 健康状态。
3. 进入题库管理。
4. 搜索题库。
5. 单题录入。
6. 查询导入任务。
7. 进入知识库重建。
8. 选择 dry-run。
9. 触发任务并查看进度。

重点讲解：

- 材料目录是事实来源。
- 导入任务异步执行。
- 全量任务有并发保护。
- 幂等键避免重复提交。

## 3. 验收清单

### 3.1 账号与权限

- 普通用户可以注册、登录、退出。
- 错误密码有明确提示。
- 未登录访问受保护页面会跳转登录。
- 普通用户无法进入管理员页面。
- 管理员可以进入知识库重建和题库管理。

### 3.2 简历与 JD

- 可以上传简历。
- 可以查看简历列表。
- PDF 可以预览。
- 可以新增 JD。
- 可以按岗位和公司筛选 JD。
- 用户上传 JD 可以删除。
- 系统预置 JD 不允许删除。

### 3.3 AI 面试

- 可以创建面试。
- 第一题可以展示。
- 文本回答可以提交。
- 轮次任务可以返回下一题。
- 历史轮次可以展示。
- 可以暂停。
- 可以恢复。
- 可以结束。
- 结束后可以生成报告。

### 3.4 语音能力

- 浏览器可以请求麦克风权限。
- 可以选择麦克风。
- 可以开始录音。
- 可以自动停止并提交。
- provider 不可用时有降级提示。

### 3.5 预约能力

- 可以创建预约。
- 可以查看预约列表。
- 可以按日期查看。
- 可以取消预约。
- 可以下载日历文件。
- 到点后可以开始预约面试。

### 3.6 报告与回放

- 报告列表可查看。
- 报告详情包含总分、优势、短板、建议、维度评分、风险。
- 失败报告可以重试。
- 历史记录可查看。
- 回放可展示完整问答。

### 3.7 题库与编程练习

- 可以创建题库练习。
- 可以提交答案。
- 可以结束练习。
- 可以查看练习记录。
- 可以进入编程题。
- 可以运行自测。
- 可以正式提交。
- 编译错误和用例失败有提示。

### 3.8 管理端

- 题库可分页查询。
- Markdown 上传有格式校验。
- 单题录入会触发导入。
- 知识库重建可提交 dry-run。
- 任务状态可轮询。
- 失败任务显示错误。

## 4. 常见演示问题与处理

### 注册 Network Error

检查后端端口和前端 `VITE_API_BASE`。如果后端是 `8000`，前端必须设置：

```powershell
$env:VITE_API_BASE="http://localhost:8000/api/v1"
```

### 页面没有管理员菜单

当前账号不是 admin。需要使用管理员账号或修改数据库中的用户角色。

### 面试问题偏模板

当前使用 mock provider。演示稳定性优先时这是正常表现。真实 AI 效果需要配置 OpenAI 兼容接口或 Ollama。

### 报告质量不高

报告质量受以下因素影响：

- 面试轮次数太少。
- 回答内容太短。
- 简历解析文本不足。
- JD 内容不完整。
- 使用 mock provider。

### 语音不可用

检查浏览器权限、麦克风设备、ASR/TTS provider 和 HTTPS/localhost 限制。

### 编程题运行失败

检查本机是否有 `g++`、`javac`、`java` 和 `node`。

## 5. 交付材料清单

项目交付时建议包含：

- 根目录 `README.md`。
- `docs/` 文档目录。
- `openapi/openapi.yaml`。
- `postman/` 调试集合。
- `backend/assets/material/` 材料源。
- `backend/assets/data/reports/` 构建报告。
- `scripts/` 启动和测试脚本。
- `.github/workflows/ci.yml`。
- 测试执行结果或 CI 链接。

## 6. 答辩讲解建议

建议按以下顺序介绍：

1. 项目解决的问题：求职者缺少高频、个性化、可复盘的模拟面试训练。
2. 核心闭环：简历/JD 输入、AI 追问、练习记录、报告复盘、专项训练。
3. 技术架构：React 前端、FastAPI 后端、SQLite、材料管道、provider 降级。
4. 关键难点：语音链路、异步轮次、报告生成、题库导入、编程判题。
5. 工程质量：测试脚本、CI、OpenAPI、Postman、文档、数据报告。
6. 后续规划：生产数据库、任务队列、代码沙箱、真实模型质量优化、用户画像。
