# 测试与质量说明

更新时间：2026-06-28

本文说明 AI-Interview 的本地测试命令、CI 门禁、测试范围和发布前检查。

## 1. 测试目标

测试体系覆盖：

- 后端接口和服务逻辑。
- 数据脚本。
- 前端组件、状态和构建。
- 端到端核心流程。
- 跨平台启动和测试脚本。

默认 CI 使用 mock provider，不依赖真实密钥、本地模型或 GPU。

## 2. 一键测试

Windows PowerShell：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\test-windows.ps1
```

Windows CMD：

```cmd
scripts\test-windows.cmd
```

macOS/Linux：

```bash
bash scripts/test-unix.sh
```

## 3. 测试脚本变量

| 变量 | 默认值 | 说明 |
| --- | --- | --- |
| `RUN_BACKEND_TESTS` | `1` | 是否运行后端 pytest |
| `RUN_FRONTEND_TESTS` | `1` | 是否运行前端 Vitest 和 build |
| `RUN_E2E` | `1` | 是否运行 Playwright E2E |
| `RUN_LINT` | `1` | 是否运行 Ruff 和 ESLint |
| `SKIP_INSTALL` | `0` | 是否跳过依赖安装 |
| `BACKEND_REQUIREMENTS` | `backend/requirements-ci.txt` | 后端测试依赖文件 |

只跑 lint：

```powershell
$env:RUN_BACKEND_TESTS="0"
$env:RUN_FRONTEND_TESTS="0"
$env:RUN_E2E="0"
.\scripts\test-windows.ps1
```

只跑后端测试：

```powershell
$env:RUN_FRONTEND_TESTS="0"
$env:RUN_E2E="0"
$env:RUN_LINT="0"
.\scripts\test-windows.ps1
```

## 4. 后端质量检查

安装 CI 依赖：

```bash
python -m pip install -r backend/requirements-ci.txt
```

编译检查：

```bash
python -m compileall backend/app backend/assets/scripts/data tests/backend
```

Ruff：

```bash
python -m ruff check backend tests
```

Pytest：

```bash
python -m pytest tests/backend
```

数据脚本 dry-run：

```bash
python backend/assets/scripts/data/validate_materials.py --strict
python backend/assets/scripts/data/normalize_materials.py --dry-run
python backend/assets/scripts/data/build_question_bank.py --dry-run
python backend/assets/scripts/data/build_knowledge_vectorstore.py --dry-run
```

## 5. 前端质量检查

进入 `frontend`：

```bash
npm ci
npm run lint
npm run test
npm run build
```

端到端测试：

```bash
npx playwright install --with-deps chromium
npm run e2e
```

构建时可以注入 API 地址：

```bash
VITE_API_BASE=http://localhost:18500/api/v1 npm run build
```

## 6. GitHub Actions CI

CI 文件：

```text
.github/workflows/ci.yml
```

触发条件：

- push 到 `main`。
- pull request 到 `main`。
- 手动 `workflow_dispatch`。

### 6.1 后端门禁

步骤：

1. Setup Python 3.11。
2. 安装 `backend/requirements-ci.txt`。
3. shell 脚本语法检查。
4. `compileall`。
5. 数据脚本 dry-run。
6. Ruff。
7. Pytest。

CI 环境变量：

- `AI_INTERVIEW_APP_ENV=dev`
- `AI_INTERVIEW_LLM_PROVIDER=mock`
- `AI_INTERVIEW_ASR_PROVIDER=mock`
- `AI_INTERVIEW_TTS_PROVIDER=mock`
- `AI_INTERVIEW_AUTH_ENABLE_DEV_STATIC_TOKEN=true`
- `AI_INTERVIEW_RETRIEVAL_FALLBACK_ENABLED=true`
- `AI_INTERVIEW_PROVIDER_TRUST_ENV=false`

### 6.2 前端门禁

步骤：

1. Setup Node 20。
2. `npm ci`。
3. `npm run lint`。
4. `npm run test`。
5. `npm run build`。
6. 安装 Chromium。
7. `npm run e2e`。
8. 失败时上传 Playwright report 和 test-results。

### 6.3 Windows 脚本 smoke

步骤：

1. 解析 PowerShell 脚本语法。
2. 以跳过安装和测试的方式执行 `scripts/test-windows.ps1`。

## 9. 常见 CI 失败处理

### ESLint unused vars

删除未使用导入或变量。不要通过关闭规则绕过，除非变量确实用于类型推断且无法表达。

### React Hook dependencies

优先补齐依赖数组。如果补齐导致循环，说明 callback 稳定性需要重新设计。

### Playwright 失败

查看 artifact：

- `frontend/playwright-report`
- `frontend/test-results`

确认：

- 前端是否能启动。
- API base 是否正确。
- mock 数据是否稳定。
- 页面文本是否变更导致断言失败。

### 数据脚本 dry-run 失败

优先检查材料格式、编码、路径和报告文件。不要直接跳过数据门禁。

## 10. 测试数据建议

本地测试建议使用：

- 临时 SQLite 路径。
- mock provider。
- 固定邮箱前缀。
- 小规模材料样本。

CI 中不要依赖：

- 真实 OpenAI 密钥。
- 本地 Ollama。
- GPU。
- 固定系统时区以外的隐式时间。
