# 验收与答辩材料

更新时间：2026-06-28

本文为 AI-Interview 课程项目的验收、答辩和演示准备材料，覆盖项目亮点、演示脚本、检查清单、常见问题和成员分工说明模板。

## 1. 项目一句话介绍

AI-Interview 是一个面向求职者的 AI 模拟面试与能力提升平台，支持简历和 JD 绑定、模拟问答、语音能力、面试报告、题库练习、编程练习和管理员知识库维护。

## 2. 项目价值

- 帮助用户在真实面试前进行低成本训练。
- 结合简历和岗位描述生成更贴近目标岗位的问题。
- 通过报告指出回答优势和薄弱点。
- 通过题库练习和编程练习提供后续提升路径。
- 支持管理员维护题库和知识库，便于持续扩展。
- 支持 provider 降级，保证课堂演示环境下流程可走通。

## 3. 核心功能清单

| 模块 | 功能 | 验收方式 |
| --- | --- | --- |
| 认证 | 注册、登录、刷新、退出、找回密码 | 使用测试账号完成登录闭环 |
| 简历 | 上传、解析、列表、删除 | 上传示例简历并在面试中选择 |
| JD | 新增、查询、删除、绑定 | 创建岗位并绑定面试 |
| 面试 | 创建会话、多轮问答、暂停恢复、回放 | 完成一次模拟面试 |
| 预约 | 创建预约、取消、开始、日历导出 | 创建未来预约并下载 ICS |
| 报告 | 异步生成、查看、重试 | 结束面试后查看报告 |
| 题库练习 | 创建练习、答题、结束、记录 | 完成一次选择题练习 |
| 编程练习 | 题目列表、草稿、自测、提交 | 完成一次代码提交 |
| 管理端 | provider 健康检查、题库维护、知识库重建 | 管理员触发校验或导入 |

## 5. 演示环境准备

### 5.1 检查项目目录

```text
AI-Interview/
  backend/
  frontend/
  docs/
  openapi/
  postman/
  scripts/
```

### 5.2 检查配置

- 后端 `.env` 或环境变量是否准备好。
- 前端 `VITE_API_BASE` 是否指向正确后端地址。
- SQLite 数据库是否存在。
- 材料目录是否完整。
- provider 使用 mock 还是真实服务。

### 5.3 启动顺序

1. 启动后端。
2. 确认后端健康接口和 Swagger 可访问。
3. 启动前端。
4. 打开浏览器访问前端页面。
5. 登录测试账号。
6. 检查管理员页面 provider 状态。

## 6. 推荐演示脚本

### 6.1 开场说明

可以按以下结构介绍：

1. 项目背景：求职者需要低成本、可复盘的面试训练。
2. 核心目标：把模拟面试、报告和练习串成闭环。
3. 技术路线：前后端分离、FastAPI、React、本地数据和 AI provider。
4. 演示路径：登录、简历、JD、面试、报告、练习、管理端。

### 6.2 登录与首页

演示步骤：

1. 打开系统首页。
2. 使用测试账号登录。
3. 展示首页功能入口。
4. 说明 access token 和 refresh token 保证登录状态。

讲解重点：

- 用户数据隔离。
- 登录保护路由。
- 统一错误提示。

### 6.3 简历上传

演示步骤：

1. 进入简历管理页面。
2. 上传一份示例简历。
3. 展示简历列表。
4. 说明后续面试会使用简历文本作为上下文。

讲解重点：

- 文件上传使用 `multipart/form-data`。
- 后端保存解析文本。
- 历史面试引用简历时需要保护。

### 6.4 JD 管理

演示步骤：

1. 进入岗位管理页面。
2. 新增一个 Java 后端或 Web 前端 JD。
3. 展示 JD 列表。
4. 说明 JD 会影响问题生成和报告评价。

讲解重点：

- JD 是面试个性化的重要输入。
- 面试创建时保存 JD 快照，保证历史报告可复现。

### 6.5 创建面试

演示步骤：

1. 进入面试准备页面。
2. 选择简历、JD、岗位方向、难度和题型。
3. 创建面试。
4. 展示第一题。

讲解重点：

- 会话记录写入 `interview_sessions`。
- 问答轮次写入 `interview_turns`。
- 长耗时生成使用异步任务。

### 6.6 多轮问答

演示步骤：

1. 输入自我介绍答案。
2. 提交后等待下一题。
3. 再回答一到两轮技术或项目问题。
4. 展示处理中的 loading 状态。

讲解重点：

- 后端会创建 turn job。
- worker 负责检索、生成和降级。
- 前端轮询任务结果。
- `pipeline_meta` 记录 provider 信息和 trace ID。

### 6.7 暂停与恢复

演示步骤：

1. 点击暂停面试。
2. 返回暂停列表。
3. 恢复面试。
4. 确认历史问答仍然存在。

讲解重点：

- 会话状态持久化。
- 刷新页面不丢失进度。
- 用户可以分阶段训练。

### 6.8 结束与报告

演示步骤：

1. 点击结束面试。
2. 进入报告列表或报告详情。
3. 展示生成中状态。
4. 等待报告完成。
5. 展示优势、薄弱点和建议。

讲解重点：

- 报告由异步 worker 生成。
- LLM 不可用时可回退规则报告。
- 报告用于指导后续练习。

### 6.9 题库练习

演示步骤：

1. 进入题库练习准备页。
2. 选择题目范围。
3. 完成几道选择题。
4. 查看练习结果和记录。

讲解重点：

- 练习题来自材料和构建脚本。
- 会话保存题目快照。
- 练习记录不会因题库更新而失真。

### 6.10 编程练习

演示步骤：

1. 进入编程练习列表。
2. 选择一道简单题。
3. 编写或修改初始代码。
4. 运行自测。
5. 提交并查看结果。

讲解重点：

- 编程题来自 JSON 材料。
- 草稿和提交分开保存。
- 本地执行适合演示，不等同生产沙箱。

### 6.11 管理员页面

演示步骤：

1. 使用管理员账号登录。
2. 打开 provider 健康检查。
3. 查看题库管理或导入任务。
4. 触发材料校验或 dry-run。

讲解重点：

- 管理员路由有角色限制。
- 材料导入是异步任务。
- 报告目录保留执行结果。

## 9. 答辩讲解结构

### 9.1 背景与问题

- 面试准备成本高。
- 普通题库缺少针对简历和 JD 的个性化。
- 用户缺少可复盘的训练记录。
- 单纯聊天工具难以形成持续练习闭环。

### 9.2 解决方案

- 用简历和 JD 构造面试上下文。
- 用多轮问答模拟真实面试。
- 用报告反馈用户表现。
- 用题库和编程练习承接后续提升。
- 用管理员工具持续维护材料。

### 9.3 技术实现

- 前端负责页面、状态和交互。
- 后端负责业务编排、鉴权和数据持久化。
- 数据脚本负责材料校验、归一化和知识库构建。
- provider 层隔离真实模型和 mock 实现。
- 异步 worker 处理耗时任务。

### 9.4 项目亮点

- 功能闭环完整。
- 数据模型覆盖真实面试训练场景。
- 支持降级，演示稳定性更高。
- 文档、OpenAPI、Postman 和测试材料较完整。
- 管理端让题库和知识库可持续维护。

## 10. 常见答辩问题

### 10.1 为什么选择 SQLite

SQLite 部署简单，适合课程项目、本地演示和 MVP。系统的数据访问集中在 repository 层，后续可以迁移到 PostgreSQL。

### 10.2 真实模型不可用怎么办

系统支持 mock provider 和规则回退。即使没有真实模型，也可以展示注册、上传、面试、报告、练习和管理端完整流程。

### 10.3 如何保证用户数据隔离

主要业务表都带有 `user_id`，后端接口根据当前登录用户过滤。前端路由保护只是体验层，真正的隔离在后端。

### 10.4 为什么报告要异步生成

报告需要汇总简历、JD 和多轮问答，并可能调用 LLM。异步生成可以避免请求超时，也便于展示生成中、失败和重试状态。

### 10.5 知识库如何更新

管理员或开发者修改材料后，执行材料校验、归一化、题库构建和向量库构建脚本。构建报告保存在 reports 目录，便于排查。

### 10.6 编程练习安全吗

当前实现适合本地可信演示。若面向公网，需要独立沙箱、资源限制、网络隔离和更严格的执行审计。

### 10.7 项目如何扩展

可以从以下方向扩展：

- PostgreSQL 替代 SQLite。
- Redis 或任务队列处理异步任务。
- 对接更多模型 provider。
- 增加企业题库和岗位画像。
- 增加用户学习计划。
- 增强编程题判题沙箱。

## 12. 风险与改进方向

| 风险 | 当前处理 | 后续改进 |
| --- | --- | --- |
| 模型服务不稳定 | mock 和规则回退 | provider 重试、缓存、监控 |
| SQLite 并发能力有限 | 面向本地演示 | 迁移 PostgreSQL |
| 编程执行沙箱不足 | 本地可信环境使用 | 引入容器隔离 |
| 材料质量影响回答质量 | 校验和归一化脚本 | 增加人工审核和评估集 |
| 报告生成耗时 | 异步 worker | 队列化和进度事件 |

## 13. 演示失败应急方案

### 13.1 后端启动失败

- 检查端口是否占用。
- 检查 Python 依赖是否安装。
- 检查 `.env` 配置。
- 查看后端控制台错误。

### 13.2 前端无法访问后端

- 检查 `VITE_API_BASE`。
- 检查 CORS 配置。
- 检查后端端口。
- 打开浏览器开发者工具查看请求。

### 13.3 模型调用失败

- 切换 mock provider。
- 展示 provider 健康检查页面。
- 说明系统降级策略。
- 使用预先生成的数据演示报告。

### 13.4 报告未生成

- 查看报告状态。
- 点击重试。
- 查看后端日志。
- 使用历史报告演示详情页。

### 13.5 数据为空

- 执行数据初始化脚本。
- 检查材料目录。
- 使用预置演示账号。
- 导入备用材料。
