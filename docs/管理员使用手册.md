# 管理员使用手册

更新时间：2026-06-28

本文面向管理员，说明 AI-Interview 后台管理页面、材料维护、题库管理、知识库重建和健康检查流程。

## 1. 管理员入口

管理员登录后，左侧菜单会出现：

- 知识库重建：`/admin/imports`
- 题库管理：`/admin/questions`

只有 `role=admin` 的账号能访问这些页面。普通用户访问会被重定向到 `/overview`。

## 2. 知识库重建

知识库重建页面用于触发后端材料处理流水线。该页面适用于以下场景：

- 新增或修改 Java/Web 知识库材料。
- 新增或修改题库材料。
- 更换 embedding 模型后重建向量库。
- 演示前重新生成干净数据。
- 排查检索质量问题。

### 2.1 表单字段

- 重建模式：
  - 全量重建：重新构建指定岗位域的知识库和向量库。
  - 增量重建：按现有脚本执行增量构建。
- 岗位范围：
  - Java。
  - Web。
- 模型：
  - 分块模型固定为 `qwen2.5:7b`。
  - 嵌入模型固定为 `nomic-embed-text`。
- dry-run：
  - 仅执行校验和模拟构建，不落地正式产物。

### 2.2 任务状态

提交后页面会轮询任务状态：

- `PENDING`：任务已创建，等待执行。
- `RUNNING`：任务执行中。
- `SUCCESS`：任务成功。
- `FAILED`：任务失败。
- `PARTIAL_SUCCESS`：部分成功。

同时展示：

- 任务 ID。
- 当前阶段。
- 进度百分比。
- 错误信息。
- 失败后按上次参数重试。

### 2.3 后端流水线

完整导入链路包括：

1. `validate`：校验材料。
2. `normalize`：归一化材料。
3. `chunking`：知识分块。
4. `question_bank`：构建题库。
5. `embedding`：构建向量库。
6. `evaluating`：检索评估。
7. `report`：输出报告。

题库专用任务只执行校验、归一化和题库构建。

### 2.4 幂等与并发

后端支持幂等键。相同幂等键命中时会返回历史任务。全量重建任务运行中时，后端会拒绝新的全量任务，避免多个全量任务同时写数据目录。

## 3. 题库管理

题库管理页面用于维护 Java/Web 面试练习题。

### 3.1 查询题库

支持按以下条件查询：

- 岗位域：Java、Web。
- 类别：技术、项目、场景、行为。
- 关键词。
- 页码和每页数量。

列表展示：

- 题目 ID。
- 岗位域。
- 类别。
- 标题。
- 题干。
- 解析。
- 来源路径。
- 更新时间。

### 3.2 上传 Markdown 题库

上传要求：

- 文件后缀必须为 `.md`。
- 文件名不能包含路径穿越。
- 内容必须符合题库 Markdown 结构。

Java 题库格式：

```markdown
## 第 1 题：问题标题

### 题干

请说明 Java 线程池的核心参数，以及这些参数如何影响任务执行。

### 类别

技术

### 解析

解析应覆盖 corePoolSize、maximumPoolSize、queue、keepAliveTime、拒绝策略和实际调优场景。
```

Web 题库格式：

```markdown
### 第 1 题：问题标题

#### 题干

请结合一个前端项目说明你如何定位并优化首屏加载慢的问题。

#### 类别

项目

#### 解析

解析应覆盖指标采集、资源体积分析、网络瀑布流、代码分割、缓存策略和上线验证。
```

类别只允许：

- 技术。
- 项目。
- 场景。
- 行为。

上传成功后，后端会写入材料目录并触发题库增量导入。

### 3.3 单题录入

单题录入字段：

- 岗位域。
- 类别。
- 标题。
- 题干。
- 解析。
- 来源备注。

后端会按当前日期生成 `admin-added-YYYYMMDD.md`，写入标准 Markdown，并触发题库增量导入。

## 4. Provider 健康检查

管理端和首页会调用：

- `GET /api/v1/admin/providers/health`

健康检查覆盖：

- ASR。
- LLM。
- TTS。
- embedding/检索相关 provider。

返回状态：

- `UP`：可用。
- `DOWN`：不可用。
- `DEGRADED`：降级。

如果 provider 不可用，面试主链路会尽量使用模板、mock 或规则回退，确保演示流程不断。

## 5. 材料目录维护

管理员维护材料时应遵循固定目录：

- Java 面试题：`backend/assets/material/java/java-interview/`
- Java 知识库：`backend/assets/material/java/java-knowledge/`
- Web 面试题：`backend/assets/material/web/interview.md`
- Web 知识库：`backend/assets/material/web/knowledge.md`
- 编程练习题：`backend/assets/material/coding/programming_practice_questions.json`

不要把临时文件、备份文件或非 Markdown 题库文件放入材料目录，否则可能导致校验失败或导入重复。

## 6. 管理员常用接口

- `POST /api/v1/admin/imports/materials`
- `GET /api/v1/admin/imports/materials/{task_id}`
- `GET /api/v1/admin/providers/health`
- `GET /api/v1/practice/questions`
- `POST /api/v1/practice/questions/upload`
- `POST /api/v1/practice/questions`
- `GET /api/v1/practice/questions/import-tasks/{task_id}`

写接口建议传入 `X-Idempotency-Key`，避免浏览器重复提交或网络重试造成重复任务。

## 7. 管理操作前检查

执行知识库或题库维护前，建议检查：

- 当前是否有人正在使用演示环境。
- 是否已有全量重建任务在运行。
- 材料文件是否符合格式。
- 本地模型和 embedding 服务是否可用。
- 是否已备份 `backend/assets/data/`。
- 是否需要先执行 dry-run。

## 8. 常见问题

### 全量重建被拒绝

说明已有全量任务在运行。等待当前任务结束，或查询任务状态确认是否卡住。

### Markdown 上传失败

检查文件名、后缀、题目 heading、题干/类别/解析顺序和类别枚举。

### 导入任务失败

查看任务状态中的 `last_error`，再检查：

- Ollama 是否可用。
- Chroma 目录权限是否正常。
- Python 依赖是否安装完整。
- 材料文件是否有编码或格式问题。
