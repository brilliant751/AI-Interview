# Agent1 需求与接口设计：面试难度入模 + 语音题目可视化与重播

## 1. 需求摘要
- 用户在创建面试时选择的难度（`easy`/`medium`/`hard`）必须在整个面试过程中生效，并显式传递给大模型用于控制追问强度。
- 面试过程中语音题目区不再显示浏览器默认播放条，改为“分贝柱形”播放可视化。
- 题目文本在语音播放时以流式方式逐步显示在播放区下方。
- 语音播放结束后提供“重播”按钮，支持重复播放当前题目语音。

## 2. 范围定义（In/Out）
### In Scope
- 后端问题生成链路将 `difficulty` 纳入提示词上下文。
- 前端面试页语音题目区 UI 重构（隐藏原生 `controls`，改为可视化 + 文本流式展示 + 重播）。
- 面试详情区展示当前难度。
- 相关单元测试更新。

### Out of Scope
- 新增难度档位（仅保留 `easy`/`medium`/`hard`）。
- 调整数据库字段或迁移（现有字段可复用）。
- 变更 TTS 服务接口。

## 3. 接口定义
- 现有接口字段保持不变：
  - `POST /interviews` 请求体 `difficulty`
  - `GET /interviews/{id}/status` 响应体 `difficulty`
- 本次仅变更“服务端内部消费方式”，不新增/删除 API 字段。

## 4. 数据模型草案
- 复用既有会话字段：`difficulty`（枚举：`easy | medium | hard`，默认 `medium`）。
- 新增内部映射（非持久化）：
  - `easy -> 基础概念/低压追问`
  - `medium -> 标准深度/兼顾实现与取舍`
  - `hard -> 深入细节/高压验证与反问`

## 5. 前端交互与状态约束
- 语音题目区状态：
  - `idle`：未播放/待播放
  - `playing`：播放中，分贝柱动态，作答按钮保持禁止
  - `ended`：播放结束，显示可点击“重播”
- 文本流式显示：
  - 每次题目切换后从 0 字开始递增渲染。
  - 若题目变化或重播，不重置题目文本内容，仅在新题时重置。
- 重播按钮：
  - 仅在 `ended` 且存在 `tts_audio_url` 时可用。
  - 点击后从头播放当前题目音频。

## 6. 非功能需求
- 性能：流式渲染采用轻量定时器，避免大颗粒重渲染。
- 可用性：语音不可用时降级展示完整文本。
- 兼容性：依赖标准 `HTMLAudioElement` 事件（`play/pause/ended/error`）。
- 可观测性：沿用现有日志与错误提示，不新增埋点。

## 7. 验收标准（Given-When-Then）
1. Given 创建会话时选择 `hard`，When 提交任意一轮回答，Then 大模型提示词中包含 `hard` 及其对应提问策略描述。
2. Given 输出模式为语音且存在 `tts_audio_url`，When 进入题目区，Then 页面不显示浏览器默认播放条，仅显示分贝柱。
3. Given 语音开始播放，When 播放进行中，Then 题目文本按流式逐步追加显示。
4. Given 语音播放结束，When 用户查看题目区，Then 可见“重播题目语音”按钮并可触发重播。
5. Given 面试详情区，When 会话已加载，Then 显示当前难度标签（简单/中等/困难）。

## 8. 风险与待确认项
- P1：流式渲染速度过快/过慢可能影响阅读体验。建议默认 35ms/字并允许后续配置化。
- P1：不同浏览器自动播放策略不同，首次自动播放可能被拦截，需保持手动重播兜底。
- P2：分贝柱为视觉拟态，不反映真实音频能量；若后续需真实频谱需引入 Web Audio 分析器。
