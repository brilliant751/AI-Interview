# Agent1 需求说明：简历 + JD 联合解析提问（2026-05-06）

## 目标
- 面试问答链路在已绑定 JD 的场景下，需要同时利用“简历内容 + JD 内容”生成更有针对性的追问。

## 范围
- 不新增接口，仅调整现有提问生成策略。
- 保持现有会话创建/轮次提交协议不变。

## 输入
- 候选人当前回答 `answer`。
- 会话绑定的 JD 快照（如存在）。
- 简历解析文本（已在会话流程中可访问）。
- RAG 检索参考片段。

## 输出
- 一句中文追问问题，满足：
  - 与回答直接相关；
  - 优先覆盖 JD 关键要求与简历经历之间的匹配验证；
  - 避免泛化、避免多问。

## 验收标准（AC）
1. 绑定 JD 时，模型提示词中必须包含 JD 片段与简历片段，不仅是标题。
2. 返回问题在人工抽样中可明显体现“岗位要求校验”倾向。
3. 不绑定 JD 时行为不退化，仍可正常生成追问。
4. 不改动 API schema，不影响前端交互契约。

## 风险
- 片段过长可能导致上下文噪声上升。
- 引用片段顺序不当会弱化 JD 约束。

## 非目标
- 本次不改数据库结构、不新增接口字段。
- 本次不调整评分模型与阶段流转规则。
